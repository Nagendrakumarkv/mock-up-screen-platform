# OSCAR Workbench — Angular 8 Migration Plan

## Overview

Convert the existing monolithic `OSCAR_Coding_Workbench_LightTheme.html` into a modular, production-ready **Angular 8** project. Every major UI section becomes a reusable Angular component. A JSON-driven configuration object controls all form fields across the 4 coding tabs (ICD, ICD Gaps, CPT, CPT Gaps), making the UI fully data-driven.

---

## Proposed Component Tree

```
AppComponent (root shell)
├── TopbarComponent           ← OSCAR logo, patient info, badges, right buttons
├── WorkspaceComponent        ← 3-column resizable layout host
│   ├── EncounterPanelComponent     ← LEFT: encounter list + search
│   ├── CenterPanelComponent        ← CENTER: PDF viewer + Claims table
│   │   ├── PdfViewerComponent
│   │   └── ClaimsTableComponent
│   └── CodingPanelComponent        ← RIGHT: coding workbench
│       ├── CodingTabsComponent     ← Tab bar (ICD / ICD Gaps / CPT / CPT Gaps)
│       └── CodingTabContentComponent  ← Dynamic add-form + entries list (one per tab)
│           └── OppLookupComponent  ← Reusable Opportunity Type hierarchical dropdown
└── ToastComponent            ← Global toast notifications
```

**Shared / Utility:**
- `CodingEntryRowComponent` — single row display for ICD / Gap / CPT / CPT Gap entries (with inline edit toggle)
- `OppLookupComponent` — standalone searchable hierarchical dropdown

## Key Design: Dynamic Tab Field Configuration

Each tab's form fields (both Add and Edit) are driven by a `TabFieldConfig[]` array.

### `tab-fields.config.ts` (example)

```typescript
export const ICD_FIELD_CONFIG: TabFieldConfig[] = [
  { key: 'icd',  label: 'ICD-10',  type: 'text',   placeholder: 'E11.65', cols: 4, listId: 'icdList' },
  { key: 'hcc',  label: 'HCC Cat', type: 'text',   placeholder: '19',     cols: 4 },
  { key: 'raf',  label: 'RAF Wt',  type: 'text',   placeholder: '0.302',  cols: 4 },
  { key: 'dos',  label: 'DOS',     type: 'text',   placeholder: '09/14/24', cols: 4 },
  { key: 'desc', label: 'Description', type: 'text', placeholder: '...', cols: 2, disabled: true },
  { key: 'src',  label: 'Evidence Source', type: 'select', options: ['PDF Page 1', 'Claims', 'Lab Result', 'Manual'], cols: 2 },
  { key: 'conf', label: 'Confidence', type: 'select', options: ['High','Medium','Low'], cols: 3 },
  { key: 'notes',label: 'Coder Notes', type: 'text', placeholder: 'Notes...', cols: 3 },
  { key: 'opp',  label: 'Opportunity Type', type: 'opp-lookup', cols: 3 },
];
```

The `CodingTabContentComponent` loops over this config to render fields dynamically using `*ngFor`.

## Proposed Changes

### Core Setup

#### [NEW] `angular.json`, `package.json`, `tsconfig.json`
Auto-generated by Angular CLI for Angular 8.

#### [NEW] `src/styles.scss`
Migrated all CSS from the HTML `<style>` block, split logically into global vars, components, and utilities.

---

### Models

#### [NEW] `tab-config.model.ts`
```typescript
export interface TabFieldConfig {
  key: string;
  label: string;
  type: 'text' | 'select' | 'opp-lookup' | 'number';
  placeholder?: string;
  options?: string[];         // for type: 'select'
  cols?: number;              // grid column span
  rows?: number;              // grid row span (if multi-row groups)
  disabled?: boolean;
  listId?: string;            // for datalist autocomplete
  editOnly?: boolean;         // field only visible in edit mode
  addOnly?: boolean;          // field only visible in add mode
}
```

#### [NEW] `coding-entry.model.ts`
Typed interfaces for `HccEntry`, `GapEntry`, `CptEntry`, `CptGapEntry`.

---

### Services

#### [NEW] `tab-config.service.ts`
Provides `getTabConfig(tabId: string): TabFieldConfig[]`. Imports from `tab-fields.config.ts`.

#### [NEW] `coding-data.service.ts`
- Holds reactive `BehaviorSubject` arrays for each data set (hccData, gapData, cptData, cptGapData).
- Exposes `add`, `update`, `remove` methods per tab.
- Used by `CodingTabContentComponent`.

#### [NEW] `toast.service.ts`
Singleton service for showing toast messages via `showToast(message: string)`.

---

### Components

#### [NEW] `TopbarComponent`
- Inputs: `patient`, `stats`, `reviewYear`
- Emits: `chartDetails`, `chartHistory`, `quickGuide`
- Contains patient chip, badges (RAF, HCCs, Gaps, CPTs), action buttons.

#### [NEW] `EncounterPanelComponent`
- Loads encounter list from `EncounterService`
- Filter by DOS/provider/ICD via search input

#### [NEW] `PdfViewerComponent`
- Page navigation, fullscreen toggle, annotation/highlight buttons

#### [NEW] `ClaimsTableComponent`
- Column selector, group-by toggle, sortable headers, filter chips
- Renders rows from `ClaimService`

#### [NEW] `CodingPanelComponent`
- Hosts tab bar + DOS tabs + coding content
- Manages fullscreen state

#### [NEW] `CodingTabContentComponent`
- **Key component**: Receives `tabConfig: TabFieldConfig[]` as `@Input`
- Dynamically renders add-form fields using `*ngFor` over the config
- Manages the list of coding entries
- Supports adding / inline editing / deleting

#### [NEW] `CodingEntryRowComponent`
- `@Input() entry`: any coding entry
- `@Input() editMode: boolean`
- `@Output() edit`, `@Output() delete`, `@Output() update`, `@Output() cancel`
- Renders both display row and inline edit form using the same tab config

#### [NEW] `OppLookupComponent`
- `@Input() value: string`
- `@Output() valueChange: EventEmitter<string>`
- Standalone, reusable hierarchical search dropdown
- Works as `[(ngModel)]` compatible control via `ControlValueAccessor`

#### [NEW] `ToastComponent`
- Subscribes to `ToastService`, auto-hides after 2.5s

---

## CSS Migration Strategy

- **Global variables**: `:root` CSS vars → `src/styles.scss`
- **Component styles**: Each component gets its own `.scss` file with scoped styles using Angular `ViewEncapsulation.None` (to preserve existing class names without refactoring all CSS)
- **No stylesheet framework** added; pure migration of existing vanilla CSS
